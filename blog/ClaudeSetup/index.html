1. Run from root

  • When you first launch Claude code, run it from your project's root directroy. Cladue works best when it can see your whole project structure. claude

2. Run /init immediately.

Creates a Claude.md file. Think of it as Claude's long-term memory. /init is starting point. Review it, refine it, make it yours.

3. CLAUDE.md is hierarchical

Global at ~/.claude/CLAUDE.md. Project-level at /CLAUDE.md. Subdirectory for module rules. Most specifics wins. /memory - Select memory

1. Project memory  Checked in at ./CLAUDE.md 
2. User memory Saved in ~/.claude/CLAUDE.md

4. Keep CLAUDE.md concise

It get prepended to everyh prompt. A bloated file eats your context window and degrates performance. Focus on what matters.

5. Structure: What, Domain, Validation

What = tech stack. Domain Context = what each part does. Validation steps = tests, linters, type checkets.
